## 1. 产品概述

### 1.1 产品定位

**AVM 项目中心** 是一款面向复杂产品研发组织的项目管理平台。它以"可视化流程引擎 + 多视图工作项 + 多维度量驾驶舱"为核心，向下承接需求/任务/缺陷/迭代/版本等业务实体，向上承载从战略目标到执行反馈的全链路协同。

AVM 的设计理念：

- **流程驱动业务**：不是"记录任务"，而是"驱动流转"。每一类工作项都通过可视化的节点流来定义其生命周期，节点的角色、必填字段、流转条件、自动化操作均在系统中显式配置。
- **配置即产品**：管理员通过零代码配置即可定义业务线、空间、工作项类型、流程、字段、权限、自动化规则，无需研发介入。
- **数据驱动决策**：度量引擎与 AI 能力贯穿全流程，从单条工作项的状态自动汇聚为团队级、项目级、组织级的实时指标。
- **开放可集成**：通过 Open API、Webhook、第三方应用市场与公司现有研发工具链（代码、CI/CD、文档、IM、审批）深度集成，避免形成新的数据孤岛。

### 1.2 目标用户与场景

| 用户类型 | 典型角色 | 核心场景 |
|---|---|---|
| 产品/规划 | 产品负责人（PO）、产品经理 | 需求池管理、Roadmap 规划、版本规划、需求评审 |
| 项目经理/PMO | Scrum Master、PMO、IPD 项目经理 | 流程编排、迭代规划、跨团队协调、风险预警、汇报 |
| 研发 | 架构师、开发工程师 | 任务领取、节点流转、缺陷修复、代码关联 |
| 测试/质量 | QA、测试工程师 | 缺陷管理、用例关联、回归跟踪 |
| 设计 | 设计师、UX | 需求关联、评审协作、设计交付物上传 |
| 管理层 | 部门负责人、高管 | 跨项目驾驶舱、研发效能度量、资源投入分析 |
| 平台管理员 | 平台管理员、配置管理员 | 空间配置、权限配置、流程模板、自动化规则 |

### 1.3 核心价值主张

1. **流程硬约束**：Stage-Gate 评审与节点流的"硬约束"机制，确保流程被严格执行，避免"流程上墙不入脑"。
2. **跨角色协同**：所有角色（产品/研发/测试/市场/销售）在一个平台、一个上下文里协作，减少信息断点。
3. **全景可观测**：从单条需求到整个产品线，任意层级可下钻、可度量、可分析。
4. **资源可调度**：人员排期与人力负荷可视化，避免资源"超载"或"闲置"。
5. **资产可沉淀**：需求、流程、模板、度量模型均可沉淀为组织级资产，越用越好。
6. **AI 提效**：在字段抽取、缺陷归类、周报生成、人力分析、风险预警等场景用 AI 替代重复劳动。

### 1.4 与飞书项目的对位关系

AVM 项目中心在能力对位上**完整覆盖**飞书项目的核心模块，并结合公司实际业务做了以下增强：

| 飞书项目模块 | AVM 对位 | AVM 增强点 |
|---|---|---|
| 工作项（需求/缺陷/任务/子任务/迭代） | 工作项管理 | 支持工作项类型自定义；预置 IPD/Agile/瀑布 三套模板 |
| 视图（表格/看板/甘特/树形/全景/度量） | 视图系统 | 视图支持更细粒度的列权限；预置跨项目全景驾驶舱 |
| 节点流 + Stage-Gate 评审 | 流程引擎 | 节点流支持父子嵌套；评审要素库可继承可定制 |
| 权限体系（5 级角色） | 权限与治理 | 增加按业务线 + 按空间双层隔离；支持字段级动态权限 |
| 度量模式 + 数据模板 | 度量与分析 | 内置研发浓度、WIP、燃尽、缺陷逃逸率等指标 |
| AI 字段 / MCP | AI 能力 | 私有化模型支持；敏感字段不出域 |
| 自动化（触发器/条件/操作） | 自动化工作流 | 跨业务线、跨空间的级联规则 |
| 轻应用 / 自定义页面 | 自定义页面与门户 | 组件市场 + 多页签导航 |
| API / Webhook | 开放平台与集成 | 提供 SDK + 沙箱环境 |
| 资源日历 / 人员排期 | 资源管理 | 与人力系统打通，支持成本核算 |
| 飞书 IM/文档/会议集成 | 协作与沟通 | 支持接入主流 IM、文档、视频会议系统 |

## 2. 用户角色与权限

### 2.1 角色体系

AVM 采用 **五级平台角色 + 项目角色** 的双重模型。平台角色决定用户在系统中"能做什么"（能力边界），项目角色决定在"具体业务线/工作项"上"是什么身份"（业务身份）。

#### 2.1.1 平台角色（自低至高）

| 角色 | 范围 | 能力 |
|---|---|---|
| **访客** | 平台级 | 只读查看被授权的空间与视图，不可创建/编辑任何工作项 |
| **成员** | 平台级 | 创建/编辑自己负责的工作项，参与被授权节点的流转 |
| **业务线管理员** | 某条业务线 | 当前业务线下的工作项编辑权限；发起评审、CSV 导出；操作除"限制节点"外的节点 |
| **空间管理员** | 某个空间 | 该空间全部工作项与空间配置的全部操作权限 |
| **租户管理员** | 整个租户 | 全部空间的管理员权限；可设置空间管理员；处理超额度人员调整 |

#### 2.1.2 项目角色（每条工作项的实例角色）

| 角色 | 范围 | 能力 |
|---|---|---|
| **创建人** | 单条工作项 | 自动拥有"实例负责人"权限；可操作除"限制节点"外的所有节点 |
| **节点负责人** | 单个节点 | 拥有该节点的编辑与流转权限 |
| **字段负责人** | 单个字段 | 拥有该字段的编辑权限（其余字段对本人只读） |
| **实例负责人** | 单条工作项 | 可操作除"限制节点"以外的所有节点和表单编辑（仅对需求生效） |
| **Feature Owner** | 一组关联工作项 | 拥有该 Feature 下全部工作项的协调权限 |

### 2.2 权限矩阵

| 操作 | 访客 | 成员 | 业务线管理员 | 空间管理员 | 租户管理员 |
|---|---|---|---|---|---|
| 查看授权空间/视图 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 创建工作项 | — | ✓ | ✓ | ✓ | ✓ |
| 编辑自己负责的工作项 | — | ✓ | ✓ | ✓ | ✓ |
| 编辑他人工作项（非限制节点） | — | — | ✓ | ✓ | ✓ |
| 操作"限制节点" | — | — | — | ✓ | ✓ |
| 发起评审 | — | — | ✓ | ✓ | ✓ |
| CSV 导出 | — | — | ✓ | ✓ | ✓ |
| 业务线配置 | — | — | ✓ | ✓ | ✓ |
| 空间配置 | — | — | — | ✓ | ✓ |
| 设置空间管理员 | — | — | — | — | ✓ |
| 删除实例 | — | — | —（除非"实例可删"开启） | ✓ | ✓ |

#### 字段级权限

每个字段独立配置授权角色。系统支持按字段类型差异化授权：

- **基础字段**（标题、描述、负责人）：默认"实例负责人"+"节点负责人"可编辑
- **业务字段**（优先级、估分、关联需求）：可独立配置可编辑角色
- **系统字段**（创建人、创建时间）：默认全员只读
- **敏感字段**（客户名称、合同金额）：可限制为"仅特定角色"可见

#### 节点级权限

每个节点独立配置授权角色。系统内置三类节点：

- **普通节点**：实例负责人 + 节点负责人可操作
- **限制节点**：仅"业务线超级管理员 + 空间管理员 + 租户管理员"可操作
- **评审节点**：仅评审通过后，节点负责人可推进

## 4. 核心功能模块

### 4.1 工作项管理

工作项是 AVM 的核心实体。一切业务活动围绕工作项展开。

#### 4.1.1 工作项类型

系统内置 5 类标准工作项，并支持管理员自定义新类型：

| 类型 | 编码 | 用途 | 默认生命周期 |
|---|---|---|---|
| 需求 | REQ | 来自市场/产品/客户的需求 | 需求池 → 评审 → 排期 → 开发 → 测试 → 验收 → 关闭 |
| 任务 | TASK | 需求拆解后的具体执行单元 | 待领取 → 进行中 → 自测 → 完成 |
| 缺陷 | BUG | 测试/用户发现的产品缺陷 | 新建 → 修复中 → 待验证 → 已关闭 |
| 迭代 | ITER | Sprint/迭代周期的容器 | 规划中 → 进行中 → 评审 → 复盘 → 关闭 |
| 版本 | REL | 发布版本/里程碑 | 规划中 → 进行中 → 集成 → 发布 → 归档 |

业务线可在此基础上扩展自定义类型（如"技术债""风险项""专利申请""客户反馈""市场活动"等）。

#### 4.1.2 工作项字段

每个工作项类型由字段组合定义。系统字段类型包括：

**基础类**
- 单行文本、多行文本、富文本（含 @ / 提及 / 表情）
- 数字、货币、百分比、评分
- 单选、多选、级联单选、级联多选
- 日期、日期时间、时间段
- 复选框、开关

**协作类**
- 人员（单选/多选，支持按部门/角色动态解析）
- 部门（单选/多选）
- 关联工作项（单/多，支持跨类型、跨空间）
- 附件（图片/视频/文档/压缩包，单文件上限 5GB）
- 飞书文档/Wiki 链接（深度嵌入预览）

**流程类**
- 状态（受节点流约束）
- 节点进度（当前节点 + 完成度）
- 节点负责人、节点角色
- 节点历史（只读，记录每次流转）
- DOD 检查清单（节点完成前必须勾选）

**度量类**
- 工作量估分（数字）
- 实际工时（数字，自动汇总）
- 优先级（紧急/重要矩阵）
- 业务影响（自定义枚举）
- 价值分（自定义算法）

**自定义类**
- 公式字段（跨字段计算）
- AI 字段（由 AI 模型生成内容）
- 自动编号（支持自定义前缀规则）

#### 4.1.3 工作项生命周期

每个工作项类型都绑定一个**节点流**。节点流定义：

- **节点**：流程步骤，含名称、角色、必填字段、入口条件、出口条件、DOD
- **连线**：节点间的流转规则（条件流转/默认流转）
- **入口/出口**：流程的起始节点和终态节点

系统预置 3 套标准节点流模板：

1. **Agile 模板**（适用于敏捷团队）：Backlog → Ready → In Progress → Code Review → Testing → Done
2. **IPD 模板**（适用于重量级产品研发）：Charter → Concept → Plan → Development → Verification → Launch → Lifecycle
3. **瀑布模板**（适用于传统行业）：需求 → 设计 → 开发 → 测试 → 验收 → 交付

#### 4.1.4 工作项关联

工作项之间通过 **关联字段** 建立关系。系统支持 5 类关联类型：

| 类型 | 关系 | 示例 |
|---|---|---|
| 父子 | 父需求 → 子任务 | Epic → Story → Task |
| 关联 | 平级互引 | 需求 A 关联需求 B |
| 阻塞 | 前后依赖 | 任务 B 阻塞任务 A |
| 重复 | 合并/关闭 | 缺陷 1 重复缺陷 2 |
| 引用 | 上下游引用 | 缺陷引用需求 |

关联关系支持反向自动展示：父需求可下钻看到所有子任务；需求详情可看到关联的缺陷、任务、文档。

关联规则可配置：

- 删除父需求时是否级联删除子项
- 子项未关闭时父项是否能关闭
- 是否允许循环关联
- 是否限制跨空间关联

### 4.2 视图系统

每个工作项集合（如一个业务线的需求池、一个迭代的任务列表）都可以通过多种视图呈现。视图的核心价值是**"按需切换，宏观到微观"**。

#### 4.2.1 表格视图

**场景**：日常运营、批量编辑、数据导出。

**能力**：
- 列自定义：用户可自由添加/移除/排序列
- 快捷编辑：双击单元格直接编辑（部分字段）
- 批量操作：勾选多条记录后批量改字段、批量分配、批量流转
- 排序与筛选：单字段排序、多字段组合排序；任意字段筛选
- 分组：按任一字段分组，支持多级分组
- 列固定：可固定关键列（如标题、状态）在最左
- 列宽自适应 / 自定义
- 保存视图：当前筛选/分组/列配置可保存为命名视图
- 导出：CSV / Excel 导出（按权限）

#### 4.2.2 看板视图

**场景**：状态流转、敏捷迭代、瓶颈识别。

**能力**：
- 列即状态：默认按"状态"字段分列
- 拖拽流转：从一列拖到另一列即可流转到对应节点
- 列内排序：拖拽调整顺序（影响排序字段）
- 卡片信息密度可调：紧凑/标准/详细
- 列汇总：每列底部显示工作项数量、估分总和
- WIP 限制：可设置每列最大在制数量，超限高亮提醒
- 列条件着色：按优先级、负责人、逾期状态着色

#### 4.2.3 甘特视图

**场景**：排期管理、进度跟踪、依赖关系可视化。

**能力**：
- 时间轴：日/周/月/季度 4 档时间粒度切换
- 任务条形：长度代表起止时间，位置代表实际进度
- 依赖连线：自动绘制前置任务依赖
- 里程碑：菱形标记
- 关键路径：高亮关键路径
- 基线对比：计划基线 vs 实际进度对比
- 今日线：红色竖线标识当前日期
- 延期标识：自动高亮延期项，支持"临期 N 天"预警
- 按层级展示：父项折叠/展开子项
- 资源视角：可切换为"按人员排期"的资源甘特

#### 4.2.4 树形视图

**场景**：多层级需求拆解、整体进度管理。

**能力**：
- 层级：基于父子关联自动构建树
- 折叠/展开：单击父节点切换
- 进度：父项自动汇总子项完成度（可配置算法）
- 估分：父项自动汇总子项估分
- 拖拽调整父子关系（需权限）
- 切换为甘特：树形顶部按钮可一键切换为"树形+甘特"混排
- 行内操作：编辑、删除、新增子项

#### 4.2.5 全景视图

**场景**：跨团队、跨项目横向协作。

**能力**：
- 多空间数据汇聚：从多个空间拉取同一类工作项
- 矩阵式布局：行为工作项，列为所属空间/项目
- 切换至其他视图：单空间内可切换至看板/甘特/度量
- 全景筛选：跨空间统一筛选
- 全景导出：导出全量数据
- 全景订阅：订阅全景视图，变更自动通知

#### 4.2.6 度量视图

**场景**：多维度数据分析、可视化驾驶舱。

**能力**（详见 4.5 节）：
- 50+ 图表类型：饼图、柱状图、折线、堆积图、桑基图、雷达图、热力图、漏斗、矩阵等
- 维度与指标自由组合
- 实时刷新
- 移动端适配
- 一键分享 / 嵌入

### 4.3 流程引擎

流程引擎是 AVM 的"心脏"。它将所有工作项的生命周期可视化、可配置化、可治理化。

#### 4.3.1 节点流

每个工作项类型关联一个**节点流**。节点流的核心元素：

- **节点**：流程的最小步骤
  - 名称、描述
  - 角色（节点负责人 + 协作角色）
  - 必填字段（DOD 字段清单）
  - 入口条件（前置节点必须完成的状态）
  - 出口条件（满足后允许流转）
  - SLA（最长时间，超时预警）
  - DOD（完成定义表单）
  - 自动化操作（流转时触发）
- **连线**：节点之间的流转路径
  - 条件流转：满足条件走 A，否则走 B
  - 默认流转：默认走向
  - 角色限制：哪些角色可发起此流转
- **入口/出口**：流程起点与终态

**操作路径**：空间配置 → 流程管理 → 节点流 → 选择工作项类型 → 进入画布拖拽编排。

#### 4.3.2 状态流转规则

节点流转时系统强制校验：

1. **角色校验**：发起人是否有该流转权限
2. **字段校验**：当前节点必填字段是否填写完整
3. **DOD 校验**：完成定义是否全部勾选
4. **条件校验**：出口条件是否满足
5. **业务规则校验**：是否触发跨工作项校验（如子项未关闭，父项不允许关闭）
6. **自动化副作用**：触发预设的自动化操作

任何一条不满足，流转被阻止并提示原因。

#### 4.3.3 Stage-Gate 评审

为支持 IPD 等重量级流程，AVM 内置 **评审引擎**，支持 3 类评审：

| 评审类型 | 用途 | 典型场景 |
|---|---|---|
| **技术评审 TR** | 技术方案、架构、设计 | TR1-TR6 |
| **决策评审 DCP** | 投资/资源决策 | CDCP、PDCP、ADCP |
| **质量评审 QR** | 测试准入/准出 | STQC、ITQC |

**评审核心要素**：

- **评审要素库**：组织级预置，可被各业务线继承定制
  - 要素名称（如"技术可行性""成本合理性""风险评估"）
  - 要素类型（评分项/勾选项/必答项）
  - 评分规则（1-5 分 / Go / Not Go / Go with Risk）
- **评审角色**：每个评审节点指定评审角色（PM、架构师、QA、财务等）
- **通过规则**：
  - 全员通过
  - 多数通过
  - 加权通过（按角色权重）
  - 一票否决（指定角色必须通过）
- **评审材料**：评审节点可关联飞书文档、附件、链接
- **评审结论**：
  - Go（通过，进入下一阶段）
  - Not Go（驳回，流转回上一节点）
  - Go with Risk（带风险通过，触发风险登记）
- **评审历史**：评审全过程留痕（要素、打分、意见、附件），形成不可篡改的证据链

**评审与流程的结合**：评审节点作为节点流中的特殊节点，其通过/驳回直接决定工作项的下一步走向。

#### 4.3.4 父子流程嵌套

支持主子流程嵌套：

- 主流程把控整体项目进度
- 子流程负责各自的业务逻辑（硬件、软件、固件可各有一条子流程）
- 父节点展开可看到子流程细节
- 父流程进度 = 子流程进度的加权汇总

这一能力特别适合**软硬一体**的复杂产品研发。

### 4.4 权限与治理

AVM 的权限设计遵循 **"最小授权 + 业务隔离 + 动态匹配"** 原则。

#### 4.4.1 业务线隔离

系统支持多业务线并行：

- 业务线 A 的工作项、数据、视图、流程模板对业务线 B 默认不可见
- 跨业务线引用需明确授权
- 业务线可配置独立的流程模板、字段集、自动化规则、度量模板

#### 4.4.2 空间隔离

空间是业务的容器。一个空间可包含多个业务线：

- 空间 A 内的所有数据对空间 B 默认不可见
- 空间管理员可设置跨空间共享视图
- 空间数据物理隔离（多租户部署时逻辑隔离）

#### 4.4.3 字段级动态权限

每个字段可独立配置授权规则：

- **静态授权**：指定具体角色/人员
- **动态授权**：基于字段值动态决定可见性
  - 示例：当"金额">100万 时，"财务负责人"可见；否则隐藏
  - 示例：当"部门"= 销售部时，销售总监可见全部字段；否则只看本部门字段
- **数据范围授权**：
  - 与当前用户相关的记录可见
  - 与当前用户部门相关的记录可见
  - 指定人员/部门下的记录可见

#### 4.4.4 行级权限

类似字段权限，工作项实例也可配置行级可见性：

- 创建人默认可见自己的全部工作项
- 实例负责人默认可见该实例全部字段
- 节点负责人默认可见流转到该节点的工作项
- 业务线/空间内默认全员可见（可配置为受限可见）

#### 4.4.5 审计日志

所有关键操作均记录审计日志：

- 工作项创建/编辑/删除/流转
- 字段修改前后值
- 权限变更
- 流程配置变更
- 自动化规则触发

日志支持按时间、用户、操作类型、工作项类型、空间范围检索。

### 4.5 度量与分析

AVM 提供从"单条工作项"到"组织全局"的多层级度量能力。

#### 4.5.1 图表配置

视图右上角一键切换至"度量模式"，可创建图表：

**步骤**：
1. 点击"添加图表"
2. 选择模板（自定义/系统模板/已有图表复用）
3. 输入图表名称
4. 选择图表类型
5. 配置维度（分组依据）
6. 配置指标（度量值）
7. 配置筛选条件
8. 保存

**图表类型**（50+）：

- 基础：饼图、柱状图、条形图、折线、面积图
- 进阶：堆积图、百分比堆积、桑基图、雷达图、热力图、漏斗图
- 专业：矩阵图、散点图、气泡图、箱线图、瀑布图、地图
- 表格：透视表、明细表

**维度与指标**：

- 维度：任意字段（单选/多选/日期/人员/部门等）
- 指标：数量、求和、平均、最大、最小、唯一计数、百分位
- 复合指标：跨字段计算（如"完成率=已完成/全部"）

#### 4.5.2 度量模板库

系统预置开箱即用的度量模板：

- **迭代度量**：燃尽图、燃尽图、WIP 图、迭代速率、估分完成率
- **缺陷度量**：缺陷趋势、缺陷密度、修复周期、逃逸率、严重等级分布
- **需求度量**：需求吞吐量、需求周期、需求价值分布、需求阻塞时长
- **资源度量**：人力负荷、投入工时分布、人员饱和度
- **效能度量**：研发浓度、Lead Time、Cycle Time、首次评审通过率
- **预测度量**：基于历史数据的迭代预测、缺陷预测

管理员可保存团队自定义模板到模板库，跨业务线复用。

#### 4.5.3 关键指标定义

AVM 内置若干行业认可的关键指标：

| 指标 | 公式 | 用途 |
|---|---|---|
| 研发浓度 | 需求总估分 /（研发周期 × 参与人数） | 衡量协作效率；行业基准：创新项目 20-30，成熟项目 35-40，平台项目 50-60 |
| Lead Time | 从需求创建到完成的时间 | 衡量端到端交付速度 |
| Cycle Time | 从开始处理到完成的时间 | 衡量开发环节效率 |
| 估分偏差 | \|实际估分 - 计划估分\| / 计划估分 | 衡量估算能力 |
| 缺陷逃逸率 | 生产环境缺陷数 / 总缺陷数 | 衡量质量 |
| 评审一次通过率 | 一次评审通过次数 / 总评审次数 | 衡量交付物质量 |
| 里程碑达成率 | 按时完成里程碑数 / 总里程碑数 | 衡量规划能力 |
| 迭代速率 | 完成估分 / 迭代周期 | 衡量产能 |

#### 4.5.4 驾驶舱

每个空间、业务线、项目可拥有多个驾驶舱页面：

- **战略驾驶舱**（高管）：跨产品线关键指标、Roadmap 全景
- **项目驾驶舱**（PM）：项目健康度、风险预警、进度偏差
- **团队驾驶舱**（团队负责人）：迭代速率、人力负荷、质量趋势
- **个人驾驶舱**（成员）：我的待办、我的工时、我的贡献

驾驶舱支持：

- 多图表组合排版
- 图表下钻（点击图表元素进入明细视图）
- 跨视图筛选联动
- 移动端适配
- 一键分享 / 嵌入到自定义页面

### 4.6 资源管理

#### 4.6.1 人员排期甘特

按人员维度展示其在多项目中的工作量分布：

- 横轴：时间（日/周/月）
- 纵轴：人员（按部门/角色分组）
- 条形：每个人员在每个项目/需求/任务的排期
- 颜色：按项目/业务线/工作项类型着色
- 负荷标识：
  - 正常（<80%）
  - 饱和（80%-100%）
  - 超载（>100%，红色高亮）
- 冲突预警：同一时段被多人分配同一角色时预警

#### 4.6.2 人力负荷分析

- 总览：团队整体负荷利用率
- 详情：每个人员的负荷曲线
- 预测：基于未来排期的负荷预测
- 建议：基于 AI 的资源调度建议（自动识别瓶颈与闲置）

#### 4.6.3 工时管理

- 工时填报：成员在工作项上填报实际工时
- 工时审批：项目负责人审核
- 工时汇总：按人/按项目/按需求多维度汇总
- 工时与估分对比：实际工时 vs 计划估分，分析估分偏差
- 成本核算：工时 × 单人成本 → 项目/需求成本

#### 4.6.4 团队日历

- 团队成员假期、会议、培训统一显示
- 与 IM 日历打通（飞书/钉钉/企业微信）
- 项目关键日期（评审、发布）自动标注

### 4.7 协作与沟通

#### 4.7.1 评论与 @ 提醒

- 工作项详情页内置评论区
- 支持文字、富文本、图片、视频、附件、表情
- @ 人员/角色/部门
- @ 触发 IM 实时通知
- 评论支持回复、点赞、解决标记
- 评论可关联到具体字段（点字段气泡即可发起评论）
- 评论可转为待办（一键创建关联任务）

#### 4.7.2 动态与活动流

每个工作项都有动态流：

- 创建/编辑/状态流转
- 字段变更（高亮前后值）
- 评论新增
- 附件上传
- 关联工作项变更
- 评审发起/通过

全局动态流：

- 我订阅的工作项的动态
- 我所在业务线/空间的关键动态
- 我 @ 我的全部动态

#### 4.7.3 通知与订阅

- **触发通知的场景**：
  - @ 我
  - 我负责的工作项状态变更
  - 我订阅的工作项更新
  - 我所在空间/项目的关键动作
  - 评审待我处理
  - 节点 SLA 即将超时/已超时
- **通知渠道**：站内、IM（飞书/钉钉/企微）、邮件、短信（关键事件）
- **通知聚合**：高频通知自动聚合为摘要，避免打扰
- **免打扰时段**：可设置夜间/周末静默

#### 4.7.4 与 IM 深度集成

- 工作项可在 IM 群中以卡片形式展示
- IM 消息可一键创建为工作项
- 工作项流转可推送为 IM 消息
- 工作项可在 IM 内被搜索/查看/编辑
- 群内讨论可一键关联到工作项

### 4.8 AI 能力

AVM 集成 AI 能力贯穿全流程。AI 服务支持**私有化部署**，敏感字段不出域。

#### 4.8.1 AI 字段

在工作项上添加 AI 字段，让 AI 自动生成内容：

- **标题生成**：从描述自动提炼标题
- **标签归类**：自动打标签（如缺陷自动分类为 UI/逻辑/性能/接口）
- **优先级建议**：基于描述与历史数据推荐优先级
- **估分建议**：基于相似历史需求推荐工作量
- **缺陷根因分析**：基于缺陷描述与代码 commit 推荐根因
- **风险评估**：基于需求复杂度、依赖关系、团队能力评估延期风险

#### 4.8.2 智能问答

在工作项详情页或全局搜索框输入自然语言问题：

- "上个迭代完成了多少需求？"
- "李雷本周负责哪些任务？"
- "哪些需求超过 SLA 还没关闭？"
- "上周新增了多少 P0 缺陷？"

AI 自动解析为度量查询，返回图表或数据。

#### 4.8.3 个人周报生成

AI 自动汇总个人本周工作：

- 完成的工作项
- 进行中的工作项
- 阻塞的问题
- 下周计划
- 个人贡献分析

支持一键调整后发送给上级。

#### 4.8.4 风险预警

AI 主动扫描项目状态，识别风险：

- 即将延期的里程碑
- 长时间未流转的工作项
- 人员超载预警
- 缺陷堆积预警
- 估分严重偏差

风险以卡片形式推送给项目负责人。

#### 4.8.5 AVM MCP（模型协作协议）

AVM 提供 MCP 协议接入，允许外部 AI Agent：

- 查询工作项状态
- 创建/更新工作项
- 触发自动化规则
- 查询度量数据
- 与 AVM 全功能对话

支持私有部署的 LLM 网关，确保数据安全。

### 4.9 自动化工作流

通过"触发器 + 条件 + 操作"三段式编排，无需代码即可实现高频流程自动化。

#### 4.9.1 触发器

| 触发类型 | 适用场景 |
|---|---|
| 工作项创建 | 新建需求/缺陷时 |
| 工作项字段变更 | 任意字段变更时 |
| 工作项状态流转 | 节点流转时 |
| 工作项关联变更 | 添加/移除关联时 |
| 定时触发 | 每日/每周/每月定时 |
| Webhook 触发 | 外部系统事件触发 |
| 评论新增 | 新增评论时 |
| 评审结果 | 评审通过/驳回时 |

#### 4.9.2 条件

- 字段条件：单字段条件、多字段组合条件（与/或/非）
- 跨工作项条件：关联工作项的字段满足
- 用户条件：当前用户/角色匹配
- 时间条件：距今 X 天内、创建超过 X 天
- 表达式条件：自定义公式

#### 4.9.3 操作

| 操作类型 | 行为 |
|---|---|
| 修改字段 | 设置字段为指定值、增量、公式 |
| 状态流转 | 触发节点流转 |
| 创建工作项 | 自动创建关联工作项 |
| 发送通知 | 站内消息、IM、邮件、短信 |
| 发送 Webhook | 推送事件到外部系统 |
| 调用 API | 调用预配置的 API |
| 发起评审 | 自动发起指定类型评审 |
| 创建/更新文档 | 在飞书文档创建或更新内容 |
| AI 字段生成 | 调用 AI 字段重新生成 |
| 添加评论 | 系统自动添加评论 |

#### 4.9.4 典型自动化场景

- **场景 1**：当缺陷被标记为 P0，自动 @ 测试负责人 + 推送 IM 群通知
- **场景 2**：当需求流转到"待验收"且超过 3 天，自动升级优先级 + 通知 PM
- **场景 3**：当任务被关闭，自动将关联的 DOD 字段汇总到父需求
- **场景 4**：每周一上午 9 点，自动生成上周团队周报 + 推送给部门负责人
- **场景 5**：当代码 commit 关联到任务且 CI 通过，自动流转任务到下一节点
- **场景 6**：当评审被驳回，自动创建风险记录 + 通知评审发起人

### 7.4 关键名词解释（补充）

- **业务线（Business Line）**：业务维度的纵向划分，用于隔离不同产品线/事业部之间的数据。
- **节点流（Node Flow）**：由节点和连线组成的可视化流程，定义工作项的生命周期。
- **实例（Instance）**：单个工作项记录（如一条具体的"需求 #1234"）。
- **业务线超级管理员**：拥有该业务线下全部工作项的"限制节点"操作权限。
- **实例负责人**：对单个工作项拥有除"限制节点"外的全部权限。
- **字段捷径**：通过可视化配置实现字段级自动化处理的能力，类似于 Excel 中的公式但更强大。
- **DOD（Definition of Done）**：完成定义，工作项流转到下一节点前必须满足的检查清单。
- **估分（Story Point）**：团队自定义的相对工作量度量单位，用于估算需求/任务的工作量。
